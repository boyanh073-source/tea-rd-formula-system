# 茶饮研发配方系统

这是一个静态前端原型，用于替代 Excel 中分散、难追溯的茶饮研发记录方式。

## 当前分区

- 原料库：拆分为茶叶原料库、香精/添加剂原料库、果汁奶小料库，并在同一页面内用页签切换。
- 拼配方案库：调用原料库中的茶叶原料，形成可复用的拼配配方。
- 成品配方库：调用茶叶原料库和果汁奶小料库；录入时可手动输入原料名称和编号，系统按关键词匹配候选。
- 其他记录：先保留入口，后续单独设计。
- 设置：维护各类下拉选项。

## 当前业务关系

1. 先录入原料库，原料按三类分库管理。
2. 拼配方案库调用茶叶原料库和香精/添加剂原料库；录入时可手动输入原料名称和编号，系统按关键词匹配候选。
3. 成品配方库调用茶叶原料库以及果汁奶小料库。
4. 成品配方中的其它配料只作为手动文本记录，不进入原料库调用统计。
5. 其他记录暂时不和成品配方建立生产关系。

## 编号和检索

- 原料、拼配方案、成品配方的编号都由使用者手动录入。
- 系统不会自动生成业务编号。
- 列表搜索支持按编号检索。
- 卡片上提供“编辑”按钮，可回填并修改当前记录的全部录入字段；修改原料或拼配编号时，会同步更新相关引用。

## 原料库字段

茶叶原料库：

- 原料名称
- 原料编号
- 一级分类：绿茶、红茶、乌龙茶、黄茶、白茶、黑茶、花茶、其他代用茶
- 二级分类
- 产地
- 风格
- 备注

香精/添加剂原料库：

- 名称
- 编号
- 分类：香原料、其他
- 二级分类
- 风格
- 备注

果汁奶小料库：

- 名称
- 供应商
- 一级分类：乳制品、果汁果酱、其他
- 二级分类

## 界面调整

- 所有手动输入框已去掉示例提示文字，保持界面简洁。
- 资料库区域改为单层简介卡片，不再同时显示右侧详细档案。
- 原料库固定为左侧录入、右侧检索结果；三个原料库不会再纵向堆叠。
- 检索结果分页显示，适合后续容纳大量记录。
- 已移除快照展示和新增记录时的快照保存。
- 卡片只保留研发过程中需要快速识别的信息。

## 拼配方案库字段

- 拼配名称
- 拼配编号
- 面向客户
- 状态：前期提案、初测通过、区域测试、成品
- 原料：初始 2 个原料行，可点击“新增原料”继续增加；每行先选茶叶或添加剂，再输入原料名称与原料编号
- 原料行可以删除，但至少保留 1 行
- 每个原料行包含添加量，按百分比手动录入
- 拼配方式

## 成品配方库字段

- 配方名称
- 面向客户
- 状态
- 版本
- 茶叶冲泡：茶叶名称、茶叶编号、茶叶量、冲泡水温、冲泡时间、热水量、冰块量
- 饮品出品：初始 2 个原料行，可点击“新增原料”继续增加；每行先选茶汤、奶/果汁或小料，再输入原料名称和用量
- SOP
- 右侧配方卡片可通过“编辑”按钮回填到表单后保存修改
- 成品配方支持多条茶汤冲泡记录，饮品出品调用茶汤时显示对应茶汤名称
- 原料、拼配方案、成品配方均支持点击右侧卡片后回填修改全部录入字段

## 设置

进入“设置”分区，可以维护这些下拉选项：

- 茶叶一级分类
- 添加剂分类
- 果汁奶小料分类
- 单位

每行一个选项，保存后会立即用于录入表单。

## 打开方式

直接用浏览器打开 `index.html`。

如果需要把系统发给同事独立使用，请发送：

```text
茶饮研发配方系统-离线单文件版.html
```

这个文件已经内嵌界面样式和功能脚本，同事不需要同时接收 `app.js` 或 `styles.css`。离线单文件版的数据保存在各自浏览器里，可以录入、修改、查询和删除，但不会和其他人的电脑自动同步。

## 本机/局域网共享数据库模式

运行 `start_server.bat`，或在当前目录执行：

```bash
python server.py
```

本机访问：

```text
http://localhost:8000/
```

同一局域网同事访问：

```text
http://你的电脑IP:8000/
```

共享数据会保存到 `data/app.sqlite3`。第一次用服务器方式打开时，如果数据库为空，系统会把当前浏览器里的本地数据写入数据库作为初始共享数据。

## 公网共享部署

当前版本已支持部署为公网网页，权限暂时全开放：任何拿到链接的人都可以查看、录入、修改和删除数据。后续可以再增加账号、角色和操作日志。

部署时需要一台公网服务器或云平台账号。推荐先用支持 Docker 和持久化磁盘的平台，例如 Render、Railway、Fly.io 或自有云服务器。

### Docker 方式

构建镜像：

```bash
docker build -t tea-rd-formula-system .
```

运行：

```bash
docker run -p 8000:8000 -v tea-rd-data:/data tea-rd-formula-system
```

访问：

```text
http://服务器公网IP:8000/
```

### Render 方式

仓库内已提供 `render.yaml`，部署时使用 Docker 环境，并挂载 `/data` 持久化磁盘。部署成功后，平台会生成公网链接，形如：

```text
https://tea-rd-formula-system.onrender.com/
```

公网模式下共享数据库路径由 `DB_PATH` 环境变量控制，默认推荐：

```text
/data/app.sqlite3
```
